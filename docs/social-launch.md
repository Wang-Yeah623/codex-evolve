## X / Twitter Style

I open-sourced a small scaffold called Codex Evolve.

The idea is simple:
Codex should not just finish tasks.
It should remember corrections, promote stable rules, and get more reliable over time.

The scaffold is built around 4 layers:

- AGENTS as a constitution
- path-scoped rules
- role-separated agents

## Long Chinese Post

我最近越来越明确一件事：

AI 工具真正能不能“越用越稳”，关键不在于它单次输出有多聪明，而在于你有没有给它搭一套能记住经验、升级规则、持续复盘的工程系统。

所以我把这套思路整理成了一个开源仓库：`Codex Evolve`

它不是 prompt 集合，也不是单次工作流脚本，而是一套最小可运行骨架，目标只有一个：

让 Codex 把“这次纠正”变成“下次默认更稳”。

核心是四层：

1. `AGENTS.md` 作为认知核心，定义目标、边界、完成标准
2. 路径作用域规则，按目录、按风险加载，而不是全局灌输
3. 规划 / 审查 / 验证 / 记忆采集的角色分工
4. correction -> observation -> learned rule 的记忆闭环

如果你也在做长期项目、多角色协作、反复纠错，这套结构应该会比“继续堆 prompt”更有价值。

仓库地址：
https://github.com/Wang-Yeah623/codex-evolve
