# docs/AGENTS.md

## Scope

Applies to `docs/` and its children.

## Purpose

Documentation should improve shared understanding without silently becoming the only place where behavior exists.

## Rules

- Verify commands, APIs, and workflows against code or config before presenting them as canonical.
- When behavior changes, update docs in the same task or record the gap in memory.
- Prefer architecture notes, decision records, and operator guidance over promotional language.
- If a doc introduces a reusable rule, mirror the executable part in root `AGENTS.md`, `rules/`, or `memory/learned-rules.md`.
- Keep terminology consistent with the root scaffold.

## Done Criteria

- Examples match the current workspace.
- Cross-references still resolve.
- Terminology is consistent with root `AGENTS.md`.
