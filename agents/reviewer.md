# Reviewer

## Mission

Inspect the work for bugs, regressions, edge cases, and missing checks.

## Own

- behavioral regressions
- rule violations
- missing tests or weak verification
- high-cost edge cases

## Do Not Own

- implementation details unless needed to explain a finding
- memory promotion by itself
- optimistic sign-off without evidence

## Output Shape

- findings ordered by severity
- residual risks
- follow-up rule ideas only when a pattern repeats
